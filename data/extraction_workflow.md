# 어휘 추출 작업 절차 (실수 방지)

## 학습된 패턴

### 실수 1: cleaned 이미지로 색상 판정
- **증상**: 녹색이 부분 제거된 잔존 픽셀을 "흐릿한 검정"으로 오인하여 추출 대상에 포함
- **사례**: image 003에서 `~ -year`, `era`, `~ -year-old`를 thin black으로 잘못 분류
- **해결**: 색상 판정은 항상 **원본 이미지**에서, cleaned는 가독성 확인용으로만 사용

### 실수 2: 인접 단어 prefix 기반 그룹화 추측
- **증상**: 같은 prefix("person...")이면 같은 굵기일 거라 추측
- **사례**: image 002에서 `personality`를 굵은 검정으로 오인
- **해결**: 모든 단어의 굵기를 인접 굵은/얇은 단어와 **1:1 직접 비교**

### 실수 3: 괄호 종류 부정확 ([] vs ())
- **증상**: 대괄호와 소괄호를 임의로 판정
- **사례**: `voyage 항해[여행](하다)` → 실제는 `항해(여행)(하다)`
- **해결**: 괄호 종류 확신 안 되면 `[?]` 마크 후 사용자 확인

### 실수 4: classifier의 검정 판정을 시각 판단으로 뒤집음
- **증상**: 자동 분류기가 검정으로 판정한 행을 "흐릿해 보여서" 직접 빼버림
- **사례**: image 003에서 `unpopular`, `honeymoon`을 시각상 녹색처럼 보여 누락
- **해결**: 자동 분류기의 GREEN 판정만 시각 재확인 가능. BLACK 판정은 1순위 신뢰. 빼려면 픽셀 값으로 재증명 필요.

### 실수 6: 동사변화표 녹색 텍스트가 위 단어 굵기 판단에 영향
- **증상**: 어떤 단어 바로 아래에 `3 X | Y | Z` 형태의 녹색 동사변화표가 있으면, 그 영역 잉크 밀도가 시각적으로 더 진해 보여 위 단어를 BOLD로 오판
- **사례**: image 004에서 `oversee` (thick=0.65, 실제 thin)를 BOLD로 오판하여 누락
- **해결**:
  - 분류기는 동사변화표를 별도 행(작은 h)으로 잡으니, 위 단어의 분류와 무관
  - 굵기 판단은 단어 자체의 stroke width만 보고, 주변 영역 시각효과 무시
  - thick < 0.70이면 thin 가능성 높음 (bold 판정 시 추가 증명 필요)

### 실수 7: 분류기가 단어+동사변화표를 GREEN 행으로 병합
- **증상**: 분류기 행 병합 로직(MAX_GAP_PX=12)이 단어와 바로 아래 변화표를 한 행으로 묶음. 변화표가 녹색이라 전체가 GREEN으로 판정됨. 실제로는 단어 부분이 thin black인데 누락
- **사례**: image 007에서 `undertake`(thin) + `3 undertake | undertook | undertaken`(green)이 h=45 GREEN 행으로 병합 → undertake 누락
- **이전 사례**: image 004 `oversee`도 비슷한 문제였음 (이때는 행 분리는 됐으나 시각 판단 실수)
- **해결**:
  1. GREEN 행 중 `h > 30`(비정상 큰 높이)이면 단어+변화표 병합 의심
  2. 해당 행의 상단 1/3 영역을 별도 시각 검사
  3. 단어가 보이면 thin black 후보로 추가

### 실수 8: 페이지 경계 침범 (다른 이미지 내용 추가)
- **증상**: 현재 이미지 하단에 잘려있는 단어가 다음 이미지에 속하는 경우, 현재 이미지 처리 중에 잘못 포함
- **사례**: image 007 처리 중 `workplace`를 추가했으나 실제 image 008 첫 단어
- **해결**: 분류기가 잡지 못한 하단 행이 있으면 다음 이미지에 속하는 것으로 간주. 추가하지 않음.

### 실수 5: "X 없고" 표현을 "제외" 으로 오해석
- **증상**: 사용자 "X 없고" 표현을 "X를 빼라"로 해석
- **사례**: image 003에서 "unpopular, honeymoon 없고"를 "빼라"로 잘못 해석
- **해결**: "없다/없고"는 우선 "목록에서 누락"으로 해석. 명시적으로 "제거하라" 표현일 때만 제거.

## 표준 절차 (각 이미지마다)

1. **cleaned 이미지 시각 스캔** (Read tool로)
   - 좌측 첫 묶음의 모든 행 식별
   
2. **자동 분류기 실행** (필수)
   ```bash
   python3 scripts/classify_rows.py "원본_이미지_경로"
   ```
   - 출력의 `★` 마크된 검정 행만이 추출 후보
   - 녹색 행은 모두 제외

3. **원본 이미지로 굵기 판정**
   - 각 검정 행에 대해 원본에서 굵기 확인
   - thick 값 참고:
     - thick ≥ 0.85: bold 가능성 높음 (메인 표제어, 제외)
     - thick 0.5-0.75: thin black 가능성 (추출 후보)
     - thick < 0.5: 옅은 thin, 의심 필요
   - 시각 검증: 인접 굵은 단어와 1:1 비교

4. **불확실 항목 표시**
   - 굵기/색상 애매: `[?]` 표시
   - 괄호 종류 애매: `[?]` 표시

5. **사용자에게 표 형태로 제시**
   - 확정 항목 + `[?]` 항목 분리 표시
   - 사용자 승인/수정 받기

6. **승인 후 저장**
   - `data/vocabulary_progress.json` 업데이트
   - `data/extraction_log.md` 행 추가
