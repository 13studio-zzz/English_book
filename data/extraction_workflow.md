# 어휘 추출 작업 절차 (실수 방지)

## 학습된 패턴

### 실수 1: cleaned 이미지로 색상 판정
- **증상**: 녹색이 부분 제거된 잔존 픽셀을 "흐릿한 검정"으로 오인하여 추출 대상에 포함
- **사례**: image 003에서 `~ -year`, `era`, `~ -year-old`를 thin black으로 잘못 분류
- **해결**: 색상 판정은 항상 **원본 이미지**에서, cleaned는 가독성 확인용으로만 사용

### 실수 2: 인접 단어 prefix 기반 그룹화 추측
- **증상**: 같은 prefix("person...")이면 같은 굵기일 거라 추측
- **사례**: image 002에서 `personality`를 굵은 검정으로 오인
- **해결**: 모든 단어의 굵기를 인접 굵은/얇은 단어와 **1:1 직접 비교**

### 실수 3: 괄호 종류 부정확 ([] vs ())
- **증상**: 대괄호와 소괄호를 임의로 판정
- **사례**: `voyage 항해[여행](하다)` → 실제는 `항해(여행)(하다)`
- **해결**: 괄호 종류 확신 안 되면 `[?]` 마크 후 사용자 확인

### 실수 4: classifier의 검정 판정을 시각 판단으로 뒤집음
- **증상**: 자동 분류기가 검정으로 판정한 행을 "흐릿해 보여서" 직접 빼버림
- **사례**: image 003에서 `unpopular`, `honeymoon`을 시각상 녹색처럼 보여 누락
- **해결**: 자동 분류기의 GREEN 판정만 시각 재확인 가능. BLACK 판정은 1순위 신뢰. 빼려면 픽셀 값으로 재증명 필요.

### 실수 5: "X 없고" 표현을 "제외" 으로 오해석
- **증상**: 사용자 "X 없고" 표현을 "X를 빼라"로 해석
- **사례**: image 003에서 "unpopular, honeymoon 없고"를 "빼라"로 잘못 해석
- **해결**: "없다/없고"는 우선 "목록에서 누락"으로 해석. 명시적으로 "제거하라" 표현일 때만 제거.

## 표준 절차 (각 이미지마다)

1. **cleaned 이미지 시각 스캔** (Read tool로)
   - 좌측 첫 묶음의 모든 행 식별
   
2. **자동 분류기 실행** (필수)
   ```bash
   python3 scripts/classify_rows.py "원본_이미지_경로"
   ```
   - 출력의 `★` 마크된 검정 행만이 추출 후보
   - 녹색 행은 모두 제외

3. **원본 이미지로 굵기 판정**
   - 각 검정 행에 대해 원본에서 굵기 확인
   - thick 값 참고:
     - thick ≥ 0.85: bold 가능성 높음 (메인 표제어, 제외)
     - thick 0.5-0.75: thin black 가능성 (추출 후보)
     - thick < 0.5: 옅은 thin, 의심 필요
   - 시각 검증: 인접 굵은 단어와 1:1 비교

4. **불확실 항목 표시**
   - 굵기/색상 애매: `[?]` 표시
   - 괄호 종류 애매: `[?]` 표시

5. **사용자에게 표 형태로 제시**
   - 확정 항목 + `[?]` 항목 분리 표시
   - 사용자 승인/수정 받기

6. **승인 후 저장**
   - `data/vocabulary_progress.json` 업데이트
   - `data/extraction_log.md` 행 추가
